# Triage Labels

The skills speak in terms of five canonical triage roles. This file maps those roles to the actual label behavior used in this repo's issue tracker.

| Canonical role | Label in our tracker | Meaning |
| --- | --- | --- |
| `needs-triage` | no label; comment-only | Maintainer needs to evaluate this issue |
| `needs-info` | no label; comment-only | Waiting on reporter for more information |
| `ready-for-agent` | `ready-for-agent` | Fully specified, ready for an AFK agent |
| `ready-for-human` | `ready-for-human` | Requires human implementation |
| `wontfix` | no label; comment-only | Will not be actioned |

When a skill mentions a role, use the corresponding behavior from this table. Do not create labels for the comment-only states unless the project owner changes this file later.
